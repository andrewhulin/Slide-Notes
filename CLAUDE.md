# CLAUDE.md

Project guide for AI assistants working on the Slide-Notes codebase.

## Project Overview

**Slide-Notes** (package name: **Ash-Notes**) is a Figma plugin that creates shapes and slides across multiple Figma editor types. It supports Figma (design), FigJam (whiteboarding), Slides (presentations), and Buzz. The plugin presents a simple UI where users enter a count, then generates editor-specific objects (rectangles, connectors, slides, or frames) accordingly.

## Repository Structure

```
Slide-Notes/
├── CLAUDE.md              # This file

```

## Tech Stack

- **TypeScript** (^5.3.2) — strict mode, ES6 target
- **Figma Plugin API** v1.0.0 — via `@figma/plugin-typings`
- **ESLint** (^8.54.0) — with `@typescript-eslint` and `@figma/eslint-plugin-figma-plugins`
- No runtime dependencies; no test framework

## Common Commands

All commands run from the `Ash-Notes/` directory:

```sh
cd Ash-Notes

npm run build      # Compile TypeScript → code.js (tsc -p tsconfig.json)
npm run watch      # Rebuild on file changes
npm run lint       # Check code with ESLint (.ts/.tsx files)
npm run lint:fix   # Auto-fix lint issues
```

## Architecture Notes

- **`code.ts`** is the single entry point. It branches on `figma.editorType` to handle each editor:
  - `figma` — creates colored rectangles, positions them horizontally
  - `figjam` — creates rounded rectangles with connectors between them
  - `slides` — creates slide objects, sets grid viewport
  - `buzz` — creates frames via the buzz API
- **`ui.html`** communicates with `code.ts` via `parent.postMessage()` / `figma.ui.onmessage`.
- The plugin closes itself after creating objects (`figma.closePlugin()`).

## Conventions

- Unused variables must be prefixed with `_` (ESLint rule).
- No production dependencies — everything is a devDependency.
- `code.js` is gitignored; always edit `code.ts` and compile.
- The `manifest.json` `editorType` array must list all supported editors.

## Gotchas

- There is no test suite. Verify changes by building (`npm run build`) and running the plugin in Figma.
- `node_modules` may not be installed. Run `npm install` in `Ash-Notes/` before building.
- The Figma plugin typings version is pinned to `*` (latest) which can cause breaking changes on fresh installs.
